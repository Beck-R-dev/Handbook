Вывод текста

Для того, чтобы вывести какой-либо текст на консоль, нужно использовать команду (statement) Console.WriteLine(), и внутри скобок в кавычках написать текст, который нужно вывести:

1
2
3
4
5
6
7
8
9
using System;

namespace ConsoleApp1 {
internal class Program {
static void Main(string[] args) {
Console.WriteLine("Hello world!");
}
}
}

В данном примере на консоль выведится текст "Hello World!":

Hello world!

*path*\ConsoleApp1\ConsoleApp1\bin\Debug\CondoleApp1.exe (process *number*) exited with code 0 (0x0).
Press any key to close this window . . .

Что касается оставшихся надписей в консоле, то это просто информация о том, что за файл был запущен, номер процесса, код ны выходе и просто текст "Нажмите на любую клавишу чтобы закрыть это окно". Единственное уточню, что вместое *path* будет полный путь до папки проекта, а вместо *number* будет какое-то число. На данным этапе, вам не нужно обращать на это внимание. Просто знайте, что это стандартная информация, которая выводится в конце выполнения вашей программы.

Чтобы вывести текст и на следующей строке, достаточно повторно использовать данную команду:

Обращаю внимание!
Хочу обратить ваше внимание на то, что текст в консоле вывелся на другой строке не из-за того, что в коде мы написали команду на другой строке, а именно из-за того, что мы саму команду написали еще раз. Другими словами, даже если написать две команды в одну строку, то результат не изменится:

Однако писать код таким образом не следует, как минимум потому, что такой код менее читаемый.

Важный момент! Вы можете подумать, что для вывода текста на другой строке, можно сделать перенос прям в коде, однако, это не так! Нельзя в коде делать перенос посреди текста. Например, если вы попробуйте написать так:

namespace ConsoleApp1 {
internal class Program {
static void Main(string[] args) {
Console.WriteLine("Hello world! 
This is second line!");
}
}
}

то такой код при компиляции выдаст ошибку. Даже до компеляции видно, что редактор подчёркивает красной волнистой линией тот участок, где был совершён неправильный перенос. Поэтому так делать не следует.

А что, если нам в консоле всё-таки нужно сделать перенос текста на следующую строку не используя повторно команду Console.WriteLine()? Для этого существуют управляющие последовательности. О них дальше.

Управляющие последовательности (escape sequences)

Сочетания символов, состоящие из обратной косой черты (\), за которой следует символ или сочетание цифр, называются управляющими последовательности (или escape-последовательностями), их также можно назвать управляющими символами. Они используются для указания определённых действий и применяются в тексте. Например, \n указывает на то, что нужно сделать перенос строки:

1
2
3
4
5
6
7
8
9
using System;

namespace ConsoleApp1 {
internal class Program {
static void Main(string[] args) {
Console.WriteLine("Line 1\nLine2");
}
}
}

Результат:

Line 1
Line 2

*path*\ConsoleApp1\ConsoleApp1\bin\Debug\CondoleApp1.exe (process *number*) exited with code 0 (0x0).
Press any key to close this window . . .

Благодаря этому управляющему символу мы можем реализовать перенос текста на следующую строку не иcпользуя повторно команду Console.WriteLine().

Кроме того, есть управляющий символ, указывающий на то, что нужно сделать табуляцию (tabulation), иначе говоря — отступ. Он пишется так: \t. Вот например:

1
2
3
4
5
6
7
8
9
using System;

namespace ConsoleApp1 {
internal class Program {
static void Main(string[] args) {
Console.WriteLine("Line 1\n\tLine2");
}
}
}

В данном примере, сразу после управляющего символа переноса строки мы добавили управляющий символ табуляции, и, соответственно, всё ровно так и произошло:

Line 1
        Line 2

*path*\ConsoleApp1\ConsoleApp1\bin\Debug\CondoleApp1.exe (process *number*) exited with code 0 (0x0).
Press any key to close this window . . .
