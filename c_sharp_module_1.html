Вывод текста

Для того, чтобы вывести какой-либо текст на консоль, нужно использовать команду Console.Write(), и внутри скобок в кавычках написать текст, который нужно вывести:

2
3
4
5
6
7
8
9
using System;

namespace ConsoleApp1 {
internal class Program {
static void Main(string[] args) {
Console.Write("Hello world!");
}
}
}

В данном примере на консоль выведится текст "Hello World!":

Hello world!
*path*\ConsoleApp1\ConsoleApp1\bin\Debug\CondoleApp1.exe (process *number*) exited with code 0 (0x0).
Press any key to close this window . . .

Конечно же, вы можете вместо "Hello world" выводить любой текст, состоящий из латинских символов. А что касается оставшихся надписей в консоли, то это просто информация о том, какой файл был запущен, номер процесса, код завершения и просто текст "Нажмите на любую клавишу, чтобы закрыть это окно". Однако уточню, что вместо *path* будет полный путь до папки проекта, а вместо *number* — какое-то число. На данном этапе вам не нужно обращать на это внимание. Просто знайте, что это стандартная информация, выводимая в конце выполнения вашей программы.

Выводить текст мы научились, но что, если нам нужно вывести ещё текст на следующей строке? Можно подумать, что для этого достаточно написать команду Console.Write() ещё раз. Однако, это сработает не так, как мы ожидаем:

То есть сам-то текст вывелся, но не на следующей строке, а сразу после предыдущего вывода. И всё же, как нам вывести текст на следующей строке? Для этого существуют управляющие последовательности. Подробнее о них — дальше.

Управляющие последовательности

Сочетания символов, состоящие из обратной косой черты (\), за которой следует символ или сочетание цифр, называются управляющими последовательности (или escape-последовательностями), их также можно назвать управляющими символами. Они используются для указания определённых действий и применяются в тексте. Например, \n указывает на то, что нужно сделать перенос строки:

2
3

5
6

using System;

namespace ConsoleApp1 {
internal class Program {
static void Main(string[] args) {
Console.Write("Line 1\nLine 2");
}
}
}

Результат:

Line 1
Line 2
*path*\ConsoleApp1\ConsoleApp1\bin\Debug\CondoleApp1.exe (process *number*) exited with code 0 (0x0).
Press any key to close this window . . .

Вот таким образом мы смогли реализовать перенос строки. Думаю стоит отметить, что перенос строки происходит ровно там, где находится \n. В нашем случае — между текстом "Line 1" и "Line 2".

Далее мы пройдём управляющий символ \t, который указывает на то, что нужно сделать табуляцию (выравнивание). Вот например:

using System;

namespace ConsoleApp1 {
internal class Program {
static void Main(string[] args) {
Console.Write("These are tabulated lines:\n");
Console.Write("\tLine 1\n");
Console.Write("\tLine 2\n");
Console.Write("\tLine 3\n");
}
}
}

Результат:

These are tabulated lines:
        Line 1
        Line 2
        Line 3

*path*\ConsoleApp1\ConsoleApp1\bin\Debug\CondoleApp1.exe (process *number*) exited with code 0 (0x0).
Press any key to close this window . . .

Как видите, перед каждым словом "Line" есть отступы, которые выравнивают текст в колонку, а в конце каждой строки — перенос. То есть всё именно так, как мы указали в коде.

У вас может возникнуть вопрос, а почему табуляция занимает ровно 8 символов? На самом деле некорректно говорить "табуляция занимает ровно 8 символов". Символ табуляции (\t) не добавляет фиксированный отступ, а переносит каретку к следующей позиции табуляции. В консоли такие позиции обычно расположены через каждые 8 символов, отсюда и выходит отступ в 8 символов. Чтобы продемонстрировать, что это именно выравнивание, а не фиксированный отступ, давайте напишем такой код:

using System;

namespace ConsoleApp1 {
internal class Program {
static void Main(string[] args) {
Console.Write("\tLine 1\n");
Console.Write("Test\tLine 2\n");
Console.Write("Another test\tLine 3\n");
}
}
}

Результат:

        Line 1
Test    Line 2
Another test    Line 3

*path*\ConsoleApp1\ConsoleApp1\bin\Debug\CondoleApp1.exe (process *number*) exited with code 0 (0x0).
Press any key to close this window . . .

Как видите, на первой строке отступ составляет 8 символов, т.к. ближайшая табуляция находится в 8 символах от начала, на второй строке — 4 символа, т.к. слово "test" занимает 4 символа и до ближайшей табуляции остаётся 4 символа, а на третьей строке отступ тоже занимает 4 символа, но уже на другой табуляции, т.к. текст "Another test" занимает 12 символов.

Думаю, на этом примере стало понятно, как именно работает выравнивание по табуляциям с помощью управляющего символа \t.

И последний управляющий символ в этом блоке — это \", который указывает на то, что нужно вывести кавычки. Например:

He said, "Hi"
*path*\ConsoleApp1\ConsoleApp1\bin\Debug\CondoleApp1.exe (process *number*) exited with code 0 (0x0).
Press any key to close this window . . .

Думаю, тут всё понятно — просто вывели кавычки. Но вы можете задаться вопросом: "А разве недостаточно просто написать кавычки в тексте?" И нет — недостаточно. Дело в том, что кавычки являются частью синтаксиса языка C#, и их лишнее использование может привести к ошибкам. Например, напишем такой код:

1

4
5
6
7
8
9
using System;

namespace ConsoleApp1 {
internal class Program {
static void Main(string[] args) {
Console.Write("He said, "Hi"");
}
}
}

Как видите, даже до компиляции редактор подсвечивает ошибку красной волнистой линией. Дело в том, что компилятор воспринимает вторую кавычку как конец строки, а после неё стоит слово "Hi" и еще две кавычки, которые нарушают синтаксис языка C#. Именно поэтому и спользуется \".

Стоит также отметить, что данное действие называется экранированием, то есть мы дали понять компилятору, что кавычка должна восприниматься просто как символ, а не как часть синтаксиса C#.

Управляющих символов существует гораздо больше, чем мы разобрали в данном блоке, однако чаще всего встречаются именно эти, поэтому нет необходимости разбирать все. Если вам интересно, вы можете самостоятельно найти информацию в интернете.
