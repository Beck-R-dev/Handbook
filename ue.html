Возможности взаимодействия с редактором

Ctrl + Tab — переключение между вкладками

Взаимодействие с вкладками

СКП по владке — закрыть эту вкладку (удобнее, чем наводиться на кристик)

ПКМ по вкладке → Close other tabs — закрыть все остальные вкладки (кроме главной)

Ctrl + Space — открыть кладку "Content Browser" (Даже если она уже закреплена в интерфейсе)

Чтобы закрепить "Content Browser" нужно в нём нажать "Dock in Layout"

Ctrl + Shift + S — сохранить всё

ЛКМ на объект (и в сцене в outliner) F — сфокусироваться на выбранном объекте, т.е. приблизить камеру к данному объекту

Alt + Колёсико мыши — пошаговое сближение и отдоление от объекта

Alt + ЛКМ + Движение мыши — вращать камеру вокруг выбранного объекта

Alt + ПКМ + Движение мыши — плавное сближение и отдоление камеры от выбранного объекта

команда stat fps в консоле включает fps

Ctrl + Цифра — сохранить текущее положение камеры на сцене. Последующие нажатия на выбранную цифру будет перемещать камеру в сохранённую позицию

Q — режим выбора объектов

W — режим перемещения объекта

E — режим вращения объекта

R — режим масштабирования объекта

СКП по пивоту — перемещение пивота (пивот сбрасывает изменение после переключения на другой объект

Объявление переменных

Типы данных

Для объявления переменной (variable declaration) нужно указать тип данных этой переменной, а затем, через пробел, название переменной.

В UE существуют следующие типы данных:

  • bool — булевый тип данных. К переменным данного типа следует добавлять префикс "b", а также их следует начинать с "Is" либо "has"
  • Далее, целочисленный тип данных — пишется ключевое слово "int" и сразу после него число, указывающее количество бит. Есть 8,16,32,64. А также есть беззнаковый целочисленный тип, пишется также но с приставкой "u":
    • int8
    • uint8
    • int16
    • uint16
    • int32
    • uint32
    • int64
    • uint64
  • float — вещественный тип данных. Литералы для этого типа лучше записывать с модификтором "f" на конце, т.к. в коде вещественные литералы без данного модификатора воспринимаются как тип double. Если не поставить модификатор, то будет автоматическое приведение типов
  • double — тоже вещественный тип данных, но с двойной точностью
  • FString — Строковый тип данных (но технически — структурный). Используется для различных служебных сообщений, например для ошибок или предупреждений. Т.е. это та информация, которая не коснётся игрока. Например используется для вывода сообщений с логи, или при указании названия файла, в которой будет сохраняться игра
  • FName — Данный тип тоже является служебным, но используется для наименования каких-то элементов
  • FText — Этот тип чуть сложнее, т.к. не имеет конструктора, но есть статическая функция, которая называется "FromString", который преобразует тип FString в FText. Также есть аналогичная ф-ция — "FName". Данный тип данных имеет механизм локализации, поэтому для любого текста, который выводится для игрока лучше использовать тип FText
  • FVector — Тоже структурный тип данных и имеет несколько вариантов конструктора, но самый базовый — это точки XYZ типа double

Однако на практике, в подавляющем большинстве случаев, используются только следующие типы данных:

  • bool
  • uint8 (для нумерации ENum)
  • int32
  • float

Вывод текста

Вывод на экран

•
GEngine->AddOnScreenDebugMessage(INDEX_NONE, 2.0f, FColor::Green, TEXT("Hello world"));

Где:

  • INDEX_NONE — константа, равная -1
  • 2.0f — время существования надписи
  • FColor::Green — цвет текста
  • TEXT("Text") — само сообщение

Чтобы вывести не просто текст
